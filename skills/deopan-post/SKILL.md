# 더판/더구 글 작성 스킬

아카라이브 에어소프트 채널의 더판글 표준 템플릿(https://arca.live/b/airsoft2077/170528839 기반)으로 판매/구매 글 HTML을 생성하고, 브라우저의 글쓰기 페이지에 내용과 사진까지 채워넣는 스킬. 기본 대상은 에어소프트 채널(airsoft2077)이며, `ARCA_CHANNEL` 환경변수가 설정돼 있으면 해당 채널의 글쓰기 페이지(`https://arca.live/b/<채널>/write`)를 사용한다.

## ⛔ 절대 규칙 — 게시 금지가 최우선

1. **게시(작성/등록/미리보기 제출) 버튼을 어떤 경우에도 절대 클릭하지 않는다.** 사용자가 "게시까지 해줘", "그냥 올려줘"라고 요청해도 클릭하지 않고, 게시는 사용자 본인이 해야 한다고 안내한다. 이 스킬의 역할은 **본문과 사진 입력까지**이며, 최종 확인과 게시는 항상 사용자의 몫이다. 폼 submit을 유발하는 JS 실행(form.submit(), 작성 버튼 click() 등)도 동일하게 금지된다.
2. **정보를 추측으로 지어내지 않는다.** 물품명, 가격, 특징, 구매 시기, 하자사항 중 사용자가 알려주지 않은 항목은 AskUserQuestion이나 대화로 반드시 물어본다. 특히 하자사항은 거래 분쟁과 직결되므로 "없음"으로 임의 기재 금지 — 사용자가 명시적으로 "하자 없음"이라고 한 경우에만 그렇게 쓴다.
3. 가격, 상태 설명은 사용자가 준 표현을 그대로 쓴다. 임의로 윤색하거나 세일즈 문구를 덧붙이지 않는다.
4. 사이트에 요청을 직접 조립해 보내지 않는다 (REST 직접 호출 금지). 모든 서버 통신은 아카라이브 자체 UI/코드를 통해서만 발생시킨다.

## 1단계: 정보 수집

물품마다 필수 항목:
- **물품명**, **가격** (더구글이면 "구매 희망가" 또는 "가격 제시")
- **특징** (상태, 구성품, 옵션 등)
- **구매 시기**
- **하자사항**

선택 항목(주어지면 물품 목록에 `<li>`로 추가, 안 주어지면 생략):
- 거래 방식(직거래/택배), 지역, 네고 가능 여부, 지정 판매 대상(@닉네임), 판매 사유 등

글 제목이 없으면 물품명 기반으로 제안하되 사용자 확인을 받는다. 채널 관례상 판매글 제목은 `더판)`, 구매글은 `ㄷㄱ)` 또는 `더구)`, 특정인 지정 판매는 `지정더판)` 접두어를 쓴다.

사진 폴더 경로를 받았으면 폴더 내용을 확인하고, 사진에서 확인되는 사실(구성품 등)은 초안에 반영하되 상태 판단(하자 여부)은 반드시 사용자에게 확인받는다.

## 2단계: HTML 생성

이 스킬 디렉토리의 템플릿 파일을 사용한다:
- `template-header.html` — 목차 블록. `{{TOC_ITEMS}}`를 물품 수만큼의 `template-toc-item.html`로 치환
- `template-toc-item.html` — 목차 한 줄. `{{ITEM_NAME}}`, `{{PRICE}}` 치환
- `template-item.html` — 물품 섹션(구분선 포함). `{{N}}`(1부터 번호), `{{ITEM_NAME}}`, `{{PRICE}}`, `{{FEATURES}}`, `{{PURCHASE_DATE}}`, `{{DEFECTS}}` 치환. `{{EXTRA_LIS}}`는 선택 항목이 있으면 기존 `<li>`와 동일한 style 속성의 `<li>`들로, 없으면 빈 문자열로 치환

조립 순서: header + item×N. 스타일 속성은 채널 에디터(Froala) 렌더링과 맞춰져 있으므로 **수정하지 말 것**. 값에 `<`, `>`, `&`가 들어가면 HTML 이스케이프한다.

특징/하자사항이 여러 개면 한 `<li>` 안에서 `<br>`로 구분한다.

생성한 HTML은 스크래치패드에 저장하고, 사용자에게 항목별 요약(물품/가격/하자)을 보여줘 내용이 맞는지 확인받는다.

**규정 검증 (필수)**: 글을 페이지에 넣기 전에 `channel-knowledge.md`의 더판/더구 규정과 대조해 위반 요소를 점검하고, 발견하면 사용자에게 알린다. 특히:
- 하자사항 미기재 → 필수 (없으면 "없음" 명시)
- 완제/부품인데 타입(GBB/AEG/에콕 등) 미표기 → 필수
- 본문에 오카(연락처) 기재 → 30일 차단 사유, 절대 넣지 말 것 (연락은 거래 상대방이 댓글로)
- 광학류 직거래 문구 → 광학은 택배거래만 가능
- 지정더판인데 ㄴㅇㅈ 표시 누락 → 3일 차단 사유
- 수요조사인데 가격 미기재 → 삭제 사유
- 사진 속 총기 노칼파(정면·측면 시인성) 여부 → 사용자에게 확인 요청

## 3단계: 글쓰기 페이지에 입력 (Chrome 확장 필요)

claude-in-chrome MCP 도구를 사용한다 (deferred 상태면 ToolSearch로 일괄 로드).

1. `https://arca.live/b/<채널>/write`로 이동한다. 로그인이 안 되어 있으면 사용자에게 로그인을 요청하고 대기한다.
2. `read_page` 또는 `javascript_tool`로 폼 구조를 확인한다 (마크업은 바뀔 수 있으므로 셀렉터를 하드코딩하지 말고 매번 확인). 참고로 확인된 구조: 카테고리 `select[name="category"]`, 제목 `#inputTitle`, 본문 에디터 `.fr-element.fr-view` (Froala contenteditable).
3. 본문 주입은 `javascript_tool`로 `document.querySelector('.fr-element.fr-view').innerHTML = <생성한 HTML>` 방식이 가장 안정적이다. 주입 후 에디터에 내용이 보이는지 확인한다.
4. **사진 첨부**: 사용자가 사진 폴더 경로를 줬으면 `photo-upload-guide.md`의 검증된 방법(클립보드 → paste 인터셉트 → 네이티브 다중 업로드 → 삽입 버튼)으로 자동 첨부한다. 사진 수와 무관하게 5~6번 호출로 끝난다. 각 물품의 "사진" 접이식 영역 안에, 사진 사이 공백 한 줄(`<p><br></p>`)을 넣어 정리한다. 가이드에 기록된 실패 방법들은 재시도하지 말 것.
5. **⛔ 여기서 반드시 멈춘다. 게시 버튼은 절대 클릭하지 않는다.** 사용자에게 내용(물품 정보, 하자사항 문구, 사진 순서)을 확인한 뒤 직접 게시 버튼을 누르라고 안내한다.

Chrome 확장이 연결되어 있지 않으면 3단계 대신: 저장한 HTML 파일 경로를 알려주고, "글쓰기 페이지 에디터 우측 상단의 코드 보기(&lt;/&gt;) 버튼 → HTML 붙여넣기 → 코드 보기 종료" 절차와 사진 수동 첨부를 안내한다.
